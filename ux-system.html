00.APostura del sistema

Qué debe transmitir · qué debe evitar

Sí · transmite
Claridad, inteligencia, control, sofisticación, modularidad, estructura, percepción premium.
Cada decisión visual debe servir a una de estas siete cualidades. Si no, se elimina.
No · se siente
Agencia · startup genérica · ecommerce · corporativo tradicional · experimento gráfico.
Evitar gradientes saturados, ilustraciones cliché, glassmorphism decorativo, slogans agresivos.
00.BVector de referencia

Identidad de sistema, no de campaña

El visual se alinea con interfaces editoriales modernas e institucionales premium: documentación técnica de empresas tecnológicas serias, reportes anuales editoriales, software B2B de alta gama. No con landing pages de agencia ni con campaña publicitaria.

Capítulo 01 · Postura

Principios UX

Siete principios que gobiernan cada decisión del sistema. Son el filtro contra el que se evalúa cualquier componente, layout o interacción nueva.

PR · 01

Diseñar antes de ejecutar

Todo elemento del sistema responde a una pregunta estructural. Nada se decora por hábito o tendencia. La arquitectura se piensa antes de pintar.

PR · 02

Modularidad obligatoria

Cada bloque es independiente, etiquetado y reutilizable. El sistema crece por composición, no por excepción.

PR · 03

Espacio antes que adorno

El espacio negativo es jerarquía. Antes de añadir un borde o un degradado, se prueba si el aire resuelve el problema.

PR · 04

Color como acento, no como ruido

La base es neutra. La marca aparece solo cuando aporta significado: identificar una unidad, marcar un acento, señalar acción.

PR · 05

Tres unidades, un sistema

Pagurai, Pagureo y Ermitaño Pep comparten estructura. Solo cambia el acento, la voz y los datos. El visitante reconoce siempre que está dentro del holding.

PR · 06

CMS-first, no decorativo

Todo contenido viene de un slot tipado. Si una sección no tiene contenido editable detrás, no existe. Esto previene el llenado decorativo.

PR · 07

Movimiento con intención

Animaciones cortas (120–360ms), curvas suaves, solo cuando refuerzan jerarquía o feedback. Nunca decorativas.

PR · 08

Pagurocorp no reemplaza · enruta

Cada unidad tiene su propia web. Pagurocorp es portal y mapa: presenta, contextualiza y delega. Nunca compite con pagurai.com, pagureo.com o canales de Ermitaño Pep.

07.CReglas críticas

Mobile no es desktop comprimido

Sí
En mobile, las cards van a 1 columna. Los heroes mantienen ratio textual pero reducen tamaños de display a H1 (44 → 32).
El ritmo vertical se mantiene; solo se acortan paddings de hero (80 → 48).
No

10.BSlot · anatomía

Cómo se visualiza un slot

Cada slot tiene un wrapper estable. El CMS llena {{ contenido }}; si está vacío, el slot se omite (no se muestra como "—" o "TBD").

required · max 80 chars
SLOT · insight.lede /insights/[slug] · zona 02

10.CComposición

El editor compone, no diseña

El admin del CMS arrastra bloques pre-tipados — hero, spotlight, grid, comparativa, quote, KPI, FAQ — y los rellena. Cada bloque define qué slots requiere y cómo se renderiza. Esto blinda la consistencia visual incluso con varios editores.

VENTAJA
Imposible romper el sistema desde el CMS.
El editor no elige tipografía, color, padding ni jerarquía. Solo elige bloque + llena slot. El sistema garantiza que toda página resultante respeta los principios.

11.BScroll behavior

Entrada de secciones

Cuando una sección entra al viewport: opacity 0 → 1 + translateY 8px → 0, en 360ms con ease-out. Stagger de 60ms entre items de un grid. Nunca animar el contenido del hero (debe estar listo al cargar).

ACCESIBILIDAD
Respetar prefers-reduced-motion.
Si el usuario tiene reducción de movimiento activada, todas las animaciones se desactivan: las entradas se vuelven instantáneas, los hovers conservan solo cambios de color (no transform, no glow).
